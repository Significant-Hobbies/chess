# Agent & Search Indexing Surfaces

The SPA shell has little server-rendered copy, so a set of static, agent- and
crawler-facing surfaces live in `public/` and are served as-is. These are
**derived** from the product description; the canonical product facts live in
[product/overview](../product/overview.md). Edit the product docs first, then
mirror into these files.

## Inventory

| File | Purpose |
| --- | --- |
| `public/llms.txt` | Short agent index (llmstxt.org format) — links to home, FAQ, machine surfaces. |
| `public/llms-full.txt` | Longer agent brief — product blurb + full surface list + fleet/contact. |
| `public/api-ai.json` | JSON inventory of public surfaces (name, version, url, llms, sitemap, robots, markdown negotiation, auth notes). |
| `public/index.md` | Markdown product brief (no JS) — the "markdown negotiation" surface. |
| `public/sitemap.xml` | `/`, `/faq`, `/index.md`, `/llms.txt`. |
| `public/robots.txt` | Allow all + sitemap + explicit allow for agent surfaces. |
| `public/_redirects` | Cloudflare Pages rewrite: `/api/ai` → `/api-ai.json` (200). |
| `public/fa7259e2e0d942f1a1267b344a75a143.txt` | IndexNow key file (Bing/Yandex URL submission). |
| `public/.well-known/security.txt` | RFC 9116 security contact (expires 2027-04-27). |
| `index.html` | `Organization` + `SoftwareApplication` JSON-LD (in the `fleet-jsonld` marked block — edit the registry, not the block). |
| `faq.html` | `FAQPage` JSON-LD + 15 crawlable Q&A. |

## Conventions

- The JSON-LD block in `index.html` is wrapped in
  `<!-- fleet-jsonld:start -->` / `<!-- fleet-jsonld:end -->` and is generated by
  a fleet tool (`apply-agent-surfaces`). **Do not hand-edit the block** — edit the
  fleet registry and re-apply, or the next run will overwrite it.
- `api-ai.json` declares `"auth": { "public": true }` — auth-walled routes are not
  listed. There are currently no auth-walled routes.
- The IndexNow key filename is the key itself (the file content is also the key).
  Don't rename it without updating the key in the IndexNow submission config.

## Adding a new public surface

1. Add the file under `public/`.
2. Add it to `sitemap.xml` if it should be crawled.
3. Add it to `llms.txt` / `llms-full.txt` / `api-ai.json` if it's an agent surface.
4. Add a `robots.txt` `Allow:` line if it needs an explicit agent allow.
5. Update [product/overview](../product/overview.md) if it changes the product
   surface area.
